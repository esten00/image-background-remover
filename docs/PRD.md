## 3. 功能需求

### 3.1 核心功能（MVP）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **图片上传** | 支持拖拽上传和点击选择，限制 JPG/PNG 格式，最大 5MB | Must |
| **去背景处理** | 调用 remove.bg API 处理图片 | Must |
| **结果预览** | 显示原图和处理后图片对比 | Must |
| **下载图片** | 提供下载按钮，保存 PNG 格式 | Must |
| **加载状态** | 处理中显示 Loading 动画 | Must |
| **错误提示** | API 失败、格式错误等友好提示 | Must |

### 3.2 非功能需求

| 需求 | 说明 |
|------|------|
| **性能** | 图片处理时间 < 10 秒（含 API 调用） |
| **响应式** | 支持桌面端和移动端 |
| **可访问性** | 基本的无障碍支持 |

## 5. API 设计

### 5.1 前端 API Route

**路径**: `POST /api/remove`

**请求**:
```typescript
{
  image: File  // FormData 格式
}
```

**响应**:
```typescript
// 成功
{
  success: true,
  imageUrl: string  // base64 或临时 URL
}

// 失败
{
  success: false,
  error: string
}
```

### 5.2 remove.bg API 调用

**端点**: `https://api.remove.bg/v1.0/removebg`

**请求**:
```bash
curl -X POST 'https://api.remove.bg/v1.0/removebg' \
  -H 'X-Api-Key: YOUR_API_KEY' \
  -F 'image_file=@/path/to/image.png' \
  -F 'size=auto'
```

**环境变量**:
```
REMOVEBG_API_KEY=xxx
```

## 8. 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| remove.bg API 失败 | 功能不可用 | 添加错误提示和重试机制 |
| 图片过大处理慢 | 用户体验差 | 前端压缩图片，设置超时 |
| Cloudflare 部署问题 | 无法上线 | 备用方案：Vercel |
| 免费额度用完 | 服务中断 | 监控用量，提示用户 |
